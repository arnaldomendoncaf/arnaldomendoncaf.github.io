Using Hugo

February 14, 2023

I decided to start logging some of my learnings in a site. So I remembered of a tool called Hugo that may help me to do it.

Hugo is a popular static site generator written in Go. In this post, I will walk you through the process of installing Hugo, configuring and publishing your first post.

Nothing better than start my tech journal with a post on how to use the tool that I’ll use to write my tech journal.

Installing Hugo#

Hugo has support for Windows, Linux, BSD and MacOS. This tutorial will focus on Linux. Said this, there are some ways to install it on linux, like package manager, binary, compiling or container. Here I’ll show you how to use the container but also how to install from source.

Installing from source#

Installing from source is not hard if you have experience with Go. You will need Go version 1.18 or later.

go install -tags extended github.com/gohugoio/hugo@latest
hugo version

It will take some time to download and compile everything.

Using as a container#

As a personal preference, I’ll use Podman, but you can run the same comands with Docker as well.

First, we will Pull the image. The official documentation points to the repository klakegg/hugo.

podman pull docker.io/klakegg/hugo
podman run --rm -ti docker.io/klakegg/hugo version

From here on, I’ll only use the container but the commands easily translate to the binary.

Create your first site#

So, let’s get to what matters. Here I’ll show you how to create and configure your first site.

First, you will need to make hugo bootstrap your site.

podman run --rm -ti -v $(pwd):/src docker.io/klakegg/hugo new site quickstart

If you are not used to Docker or Podman, the -v $(pwd):/src mounts the current directory on your host machine as the /src on your container. Hugo’s container workdirectory is the /src so the command you ran will execute in this directory.

After the container exit, you can see that Hugo created a directory called quickstart in your current directory.

If you take a look at the command output, you will see that Hugo tells you to install a new theme

Congratulations! Your new Hugo site is created in /src/xpto.

Just a few more steps and you're ready to go:

1. Download a theme into the same-named folder.
   Choose a theme from https://themes.gohugo.io/ or
   create your own with the "hugo new theme <THEMENAME>" command.
2. Perhaps you want to add some content. You can add single files
   with "hugo new <SECTIONNAME>/<FILENAME>.<FORMAT>".
3. Start the built-in live server via "hugo server".

So, take your time to visit https://themes.gohugo.io/ and choose the theme that best suits you. Here I’ll use the PaperMod theme. By their instruction, you just need to use Git to install it.

cd quickstart
git clone https://github.com/adityatelange/hugo-PaperMod themes/PaperMod --depth=1

And to use it, add the following lines to the file config.toml

theme = "PaperMod"
[params.homeInfoParams]
Title = "Hi there!"
Content = "Can be Info, links, about..."

The most important part for us, is the theme = "PaperMod", which tells Hugo to use the theme we just downloaded.

Note that the section [params.homeInfoParams] is a configuration from the PaperMod theme. Maybe the theme you chose has different configuration options.

Let’s take a look how is our page. First we need to start the builtin server.

podman run --rm -ti -v $(pwd)/quickstart:/src docker.io/klakegg/hugo server

Create your first Post#

If you read carrefully the output when you created your site, you already have a hint on how to create your post.

2. Perhaps you want to add some content. You can add single files
   with "hugo new <SECTIONNAME>/<FILENAME>.<FORMAT>".

But, how to really create your first post? Let’s say your post will be called using-hugo, it will be written in Markdown and under the section posts. So the command you will need to run is the one below:

podman run --rm -ti -v $(pwd)/quickstart:/src \
    docker.io/klakegg/hugo new posts/using-hugo.md

Content "/src/content/posts/using-hugo.md" created

Open the file quickstart/content/posts/using-hugo.md with your favorite text editor (remember that src does not exist in your host machine’s filesystem).

You will notice the following header:

---
title: "Using Hugo"
date: 2023-02-14T21:59:34+01:00
draft: true
---

The property that matter to us is the draft. As it is true, it means that our post is not ready to be published yet. So it won’t be listed in our website until this property is set to false.

By now, we wont change this header but will add some text below it. When you are ready, save the file and start the server again.

podman run --rm -ti -v $(pwd)/quickstart:/src docker.io/klakegg/hugo server

If you visit the site on the port 1313, you will notice that nothing changed since our last visit. Well, this is the job of the property draft. But you need to see how the post looks before publishing, so close the hugo server and run this command instead:

podman run --rm -ti -v $(pwd)/quickstart:/src \
    docker.io/klakegg/hugo server --buildDrafts

It tells Hugo to build the content marked as draft. Keep this server running until while you edit your post, Hugo will auto-reload the site as you edit it.

Now, if you visit the site on port 1313 again, you will see all posts marked as draft too.
