Publish your blog made with Hugo on GitHub Pages

February 15, 2023

So now you decided to publish your recently created blog on GitHub Pages. Well, this tutorial will help you do it, maybe not in the best way, as I am also a beginner, but it certainly works!

Create your GitHub Pages repository#

There is not much to say about it, you will just create a new repository on your GitHub account. The important thing is that this repository needs a special name, that is:

your_username_here.github.io

In my case, the repository name is arnaldomendoncaf.github.io. If you choose any other name it won’t work.

The repository must also be public if you don’t use GitHub Enterprise.

Set up your local copy of the repository#

Remember our first tutorial on Hugo. The blog name was quickstart, so the directory tree starts there. Initialize a repository on the public folder:

cd $(pwd)/quickstart/public
git init
git remote add origin git@github.com:${GH_USERNAME}/${GH_USERNAME}.github.io

Publish your posts#

Now that everything is set, you can publish your posts. First you will need to make sure your post is not in draft, as the pages for the draft posts will not be generated. So pick one post and edit it to change the draft property to false

sed -e '0,/^draft: true/ s//draft: false/g' \
    quickstart/content/posts/using-hugo.md

The command above will change just the first occurrence of draft: true. With that, the post is ready to be published.

To generate the page, run hugo as in

podman run --rm -ti -v $(pwd)/quickstart:/src \                              
    docker.io/klakegg/hugo

If you go back to that public folder, you will see that there are a lot of files to be committed. Go there and commit them.

cd quickstart/public
git status
git add .
git commit -m "Publishing my first content to GitHub Pages"
git push -u origin main

After pushing to GitHub, it may take some time for your blog to be updated. So test your site on ${GH_USERNAME}.github.io, if you receie a 404 status code wait a bit and try again in a couple of minutes.

Conclusion#

GitHub pages is a free way to publish your blog, and also to use git to keep control of its version.

Also remember that the HTML file for draft pages won’t be generated.
